# Onboarding 流程详解

## 触发条件

1. **首次使用**：配置文件不存在
2. **用户请求**：用户说以下任一短语：
   - "设置日程偏好"
   - "配置日程管理"
   - "初始化日程"
   - "重新设置"

---

## 流程总览

```
Step 1: 收集最近事项
Step 2: 选择分类方式
Step 3: 日对齐/周对齐时间 + 创建定时任务
Step 4: 日历设置
Step 5: 中长期目标
Step 6: 确认目标
Step 7: 完成
```

---

## 详细流程

### Step 1：收集最近事项

**问题**：
```
告诉我几件你最近在做的事情，一股脑说即可。
（这个事项清单会长期维护，随时补充和归档）
```

**处理**：
- 用户说完后，自动梳理
- 提取关键事项
- 为下一步分类展示做准备
- 创建初始事项清单（飞书文档或本地）

---

### Step 2：选择分类方式

**问题**：
```
我提供几种分类方式，你可以看到按每种方式分类后，你刚才说的事情会呈现什么样子：

【分类方式 A】MoSCoW（推荐）
- [事项1] → Must（必须做）
- [事项2] → Should（应该做）
- [事项3] → Could（可以做）
...
特点：强制取舍，简单直观

【分类方式 B】1-3-5 技术
- [事项1] → 大任务
- [事项2] → 中任务
- [事项3] → 小任务
...
特点：数量控制，每天 1 大 + 3 中 + 5 小

【分类方式 C】Eisenhower 矩阵
- [事项1] → Q1 紧急且重要
- [事项2] → Q2 不紧急但重要
...
特点：引入时间维度

你选哪个？
```

**关键点**：
- 用户能看到「如果选这个分类方式，我的事项会是什么样子」
- 给出推荐
- 用户也可以自定义分类

---

### Step 3：日对齐/周对齐时间

**问题**：
```
需要日对齐和周对齐吗？

日对齐：每天固定时间规划当天/第二天
周对齐：每周固定时间回顾和规划

- A. 都需要
- B. 只要日对齐
- C. 只要周对齐
- D. 都不需要
```

**如果需要**：
```
日对齐在什么时间？
- A. 早上 8:00（规划当天）
- B. 晚上 21:00（规划第二天）
- C. 我来说

周对齐在什么时间？
- A. 周日晚上
- B. 周一早上
- C. 我来说
```

**确定时间后立即创建定时任务**（见下方"创建定时任务"章节）

---

### Step 4：日历设置

**前置说明**：
```
日对齐和周对齐完成后，我会更新你的事项清单和日历。

关于日历，有几个问题想确认一下：
```

**问题 1：是否使用日历**
```
需要把事项同步到日历吗？

- A. 需要（推荐）
- B. 不需要，只用事项清单
```

**如果选择「需要」**：

**问题 2：颜色设置**
```
根据你选择的【分类方式】，我推荐以下配色：

[如果用户选的是 MoSCoW]
- Must（必须做）→ 红色
- Should（应该做）→ 橙色
- Could（可以做）→ 绿色
- Won't（不做）→ 灰色

[如果用户选的是 1-3-5]
- 大任务 → 红色
- 中任务 → 橙色
- 小任务 → 绿色

[如果用户选的是 Eisenhower]
- Q1 紧急且重要 → 红色
- Q2 不紧急但重要 → 紫色
- Q3 紧急不重要 → 橙色
- Q4 不重要不紧急 → 灰色

这样设置可以吗？
- A. 可以
- B. 调整（告诉我怎么改）
```

---

### Step 5：中长期目标

**问题**：
```
告诉我你的中长期目标（2-4 个）。

这是为了确保你在主线目标上花时间。周对齐时会检查是否偏离目标。

目标会出现在事项清单里，这样你就能知道你在做的事情是不是和目标有联系。

如果没有特别明确的目标，可以一股脑说自己正在关注的事情和可能的长期目标，我会帮你提取。

如果不想说目标也可以，但建议说一下。

你有哪些目标？其中哪些比较重要？（可以说「这个比较重要」来标记）
```

**处理**：
- 用户可能一股脑说很多事情
- 提取 2-4 个中长期目标
- 用户说「比较重要」的目标标记为精进型，其他为及格线

---

### Step 6：确认目标

**展示**：
```
我提取了以下目标（标 * 表示精进型目标）：

1. *[目标1]
2. *[目标2]
3. [目标3]
4. [目标4]

这个目标你看对不对？如果不对可以直接修改。
（目标随时可以在对话中更改，不用担心）
```

---

### Step 7：完成

**展示**：
```
✅ 设置完成！

- 分类方式：[用户选择的方式]
- 日对齐：每天 xx:xx
- 周对齐：每周x xx:xx
- 日历：[已启用/未启用]
- 事项清单：[文档链接]

如果上面的内容有任何需要修改的地方（改时间、改分类方式、改颜色等），随时告诉我。

当你有任何事情时，都可以直接告诉我，我会更新到事项清单里。
```

## 创建定时任务

在 Step 3 完成后（如果启用了日对齐/周对齐），使用 OpenClaw cron 创建定时任务。

### 重要说明

定时任务的消息是发给 **OpenClaw** 的，不是直接发给用户的。OpenClaw 收到消息后，需要根据消息内容执行对齐流程并与用户交互。

因此消息内容需要：
1. 告诉 OpenClaw 现在是什么对齐时间
2. 指引 OpenClaw 按照对应的流程文档执行
3. 直接让 OpenClaw 向用户发送开场消息（不要说"收到"之类的过渡语）

### 日对齐定时任务消息模板

```bash
pnpm openclaw cron add \
  --name "日对齐" \
  --cron "30 21 * * *" \
  --tz "Asia/Shanghai" \
  --message "【日对齐提醒】

现在到每日对齐时间了。

请直接向用户发送以下内容（不要说任何"收到"之类的话）：

🌙 每日对齐时间到了。

刚刚过去的一天怎么样？

然后按照 schedule-manager skill 的 references/daily-sync-flow.md 中的流程继续。

请先读取用户的配置和事项清单：
- 配置文件路径：[用户配置路径]
- 事项清单：[飞书文档链接]" \
  --channel feishu \
  --to "ou_xxx" \
  --announce
```

### 周对齐定时任务消息模板

```bash
pnpm openclaw cron add \
  --name "周对齐" \
  --cron "0 19 * * 0" \
  --tz "Asia/Shanghai" \
  --message "【周对齐提醒】

现在到每周对齐时间了。

请直接向用户发送以下内容（不要说任何"收到"之类的话）：

📅 周对齐时间到了。

先看中期目标 — 目前进展如何？

[展示用户的中长期目标表格]

然后按照 schedule-manager skill 的 references/weekly-sync-flow.md 中的流程继续。

请先读取用户的配置和事项清单：
- 配置文件路径：[用户配置路径]
- 事项清单：[飞书文档链接]" \
  --channel feishu \
  --to "ou_xxx" \
  --announce
```

### cron 表达式说明

| 对齐类型 | 时间示例 | cron 表达式 |
|----------|----------|-------------|
| 日对齐 - 早上 8:00 | 每天 8:00 | `0 8 * * *` |
| 日对齐 - 晚上 21:30 | 每天 21:30 | `30 21 * * *` |
| 周对齐 - 周日晚上 | 周日 19:00 | `0 19 * * 0` |
| 周对齐 - 周一早上 | 周一 8:00 | `0 8 * * 1` |

### 定时任务管理命令

```bash
# 列出所有定时任务
pnpm openclaw cron list

# 删除任务
pnpm openclaw cron rm <id>

# 启用/禁用任务
pnpm openclaw cron enable <id>
pnpm openclaw cron disable <id>
```

## 配置保存

完成 Onboarding 后，保存配置：

```yaml
version: "1.0"
created_at: "YYYY-MM-DD"

classification: "moscow"  # moscow / 1-3-5 / eisenhower

daily_sync:
  enabled: true
  time: "21:30"

weekly_sync:
  enabled: true
  day: "sunday"
  time: "19:00"

calendar:
  enabled: true
  color_mapping:
    "Must": -562844
    "Should": -30720
    "Could": -13318364

goals:
  - name: "目标1"
    type: "excel"
  - name: "目标2"
    type: "pass"

storage:
  feishu_doc_token: "xxx"
  feishu_calendar_id: "xxx"
  local_path: "~/.schedule-manager/tasks.md"
```

## 事项清单通用结构

创建初始事项清单：

```markdown
# 📋 [用户名]的事项清单

> 最后更新：YYYY-MM-DD HH:MM

---

## 📌 今日

**主线**：（待日对齐确定）

### 今日日程

（待填充）

---

## [根据分类模板填充结构]

---

## 🎯 目标

| 目标 | 类型 | 进展 |
|------|------|------|
| *[目标1] | 精进型 | 刚开始 |
| [目标2] | 及格线 | 刚开始 |

---

## 已完成

（完成后会记录在这里）

---

## 归档

（不再做的事项会移到这里）
```

## 飞书集成

如果用户选择使用飞书，在 Step 4 完成后：

1. **创建飞书文档**：调用 feishu-doc skill 创建事项清单
2. **设置权限**：调用 feishu-perm skill 给用户 full_access 权限
3. **获取日历 ID**：确认用户要使用的飞书日历

---

## 重新设置

如果用户说"重新设置"：

```
要重新设置吗？这会：
- 重置偏好设置
- 保留现有事项清单

确认吗？
- A. 确认，重新设置
- B. 算了，保持现状
```
