# Onboarding 流程详解

## 触发条件

1. **首次使用**：配置文件不存在
2. **用户请求**：用户说以下任一短语：
   - "设置日程偏好"
   - "配置日程管理"
   - "初始化日程"
   - "重新设置"

---

## 流程总览

```
Step 1: 收集最近事项 → 创建飞书文档
Step 2: 选择分类方式
Step 3: 日对齐/周对齐时间
Step 4: 日历颜色设置
Step 5: 中长期目标
Step 6: 确认目标 → 创建定时任务 + 测试日历写入
Step 7: 完成
```

---

## 详细流程

### Step 1：收集最近事项

**问题**：
```
告诉我几件你最近在做的事情，随意说即可，我会帮你整理。
（这个事项清单会长期维护，随时补充和归档）
```

**处理**：
- 用户说完后，自动梳理，提取关键事项，为下一步分类展示做准备

---

### Step 2：选择分类方式

**问题**：
```
我提供几种分类方式，你可以看到按每种方式分类后，你刚才说的事情会呈现什么样子：

（⚠️ 下面只是展示分类后的样子，你可以自己决定每个事项具体分到哪个类别）

【分类方式 A】MoSCoW
- [事项1] → Must（必须做）
- [事项2] → Should（应该做）
- [事项3] → Could（可以做）
...

【分类方式 B】1-3-5 技术
- [事项1] → 大任务
- [事项2] → 中任务
- [事项3] → 小任务
...

【分类方式 C】Eisenhower 矩阵
- [事项1] → Q1 紧急且重要
- [事项2] → Q2 不紧急但重要
...

除了以上方式，**如果你是一个灵感很多、又对过长的“待办事项”有压力的人**，你也可以选择在以上任意分类的基础上，再增加 **Exploration 类别**。
Exploration 引入了“无优先级”的维度，它是指但不承诺任何产出的事项。它可以被随时加入到其它类别中，且优先级可高可低。

【分类方式 D】MoSCoW + Exploration（推荐）
- [事项1] → Must（必须做）
- [事项2] → Should（应该做）
- [事项3] → Could（可以做）
- [事项4] → Exploration（无优先级）
...
特点：保护灵感，适合对待办过长有压力的人。

你选哪个？
```

---

### Step 3：日对齐/周对齐时间

**问题**：
```
每天的固定时间，我会找你回顾刚刚过去的一天、规划即将到来的一天，来帮你把精力花在主线任务上、并提醒你不要忘记临期的事项。

每周的固定时间也是类似，做一周的回顾和规划。

日对齐想在什么时间？
- A. 早上 8:00（规划当天）
- B. 晚上 21:00（规划第二天）
- C. 我来说

周对齐想在什么时间？
- A. 周日晚上
- B. 周一早上
- C. 我来说

（如果实在不想安排某个对齐，告诉我就行）
```

**关键点**：
- 不问"需不需要"，默认需要
- 弱化"可以不安排"选项（括号里提一下就行）
- 确定时间后**不要立即创建定时任务**，等 Step 6 确认后一起创建

---

### Step 4：日历颜色设置

**说明**：
```
日对齐和周对齐完成后，我会把时间安排写入飞书日历，方便你查看。

关于日历颜色，我根据你刚才说的事情，推荐按「领域」来分：
```

**问题**（基于用户事项推荐，以下为示例）：
```
- 工作 → 🔴 红色
- 学业 → 🟣 紫色
- 运动 → 🟠 橙色
- 琐事 → ⚫ 灰色

这个颜色分类可以吗？需要怎么调整？
也可以按优先级来分颜色，或者不区分颜色（用飞书默认的蓝色），或者不用飞书日历——告诉我一声就行
```

**关键点**：
- 颜色优先按**领域/项目**分类（工作、学业、内容、运动等），或者按照用户自定义的要求
- 基于用户前面说的事项来推荐

---

### Step 5：中长期目标

**问题**：
```
你有哪些中长期目标，其中哪些比较重要？

这是为了你的精力不在非主线的事上过多浪费。周对齐时会先看主线目标，为它们预留时间。

目标也会出现在事项清单里，这样你就能随时看到并记住。

你有哪些目标？如果没有特别明确的目标，可以随意说说你正在关注的事情和可能的长期目标，我来帮你提取可能的目标，再向你确认。

```

**处理**：
- 提取 2-4 个中长期目标，尽可能把相关的目标合并一个大的方向，让目标不超过4个，除非实在难以合并
- 用户说「比较重要」的目标要在后面标 `*`

---

### Step 6：确认目标

**展示**：
```
我提取了以下目标（标 * 表示重点目标）：

1. [目标1]*
2. [目标2]*
3. [目标3]

这个目标你看对不对？有不对的地方可以告诉我。
（目标之后也随时可以在对话中更改，不用担心）

---
确认后，我会创建飞书文档、测试日历写入、创建日对齐和周对齐的定时任务等，时间可能稍长，请耐心等待...
```

### **Step6确认后的操作**：（操作的具体方法见后文）

1. **飞书日历写入**
2. **创建飞书文档**
3. **创建定时任务**（日对齐 + 周对齐）
4. **更新配置文件**

---

### Step 7：完成

**展示**：
```
✅ 设置完成！

- 分类方式：[用户选择的方式]
- 日对齐：每天 xx:xx ✅ 定时任务已创建
- 周对齐：每周x xx:xx ✅ 定时任务已创建
- 日历：✅ 已测试写入成功
  - 在飞书日历左侧「我订阅的」里勾选「OpenClaw」日历即可看到
- 事项清单：[文档链接] ✅ 已给你管理权限

💡 建议：把事项清单在对话框里置顶，方便随时查看修改
（点击对话框上方的小加号 → 添加标签页 → 选择事项清单）

如果上面的内容有任何需要修改的地方，随时告诉我。

当你有任何事情时，都可以直接告诉我，我会更新到事项清单里。
```

## 飞书日历写入
- 调用 feishu-calendar skill，在 OpenClaw 日历上创建一个测试事件
- 给用户设置对 OpenClaw 的日历**编辑权限**（而不只是查看权限）
- 告诉用户：在飞书日历左侧"我订阅的"里勾选 OpenClaw 日历即可看到，如果是在手机上，点击日历界面右上角的日历小图标即可看到。

**日历写入方式**：
- 方式一（推荐）：OpenClaw 在自己日历上创建，并给用户开放编辑权限
- 方式二：用户授权写入ta自己的日历

---

## 飞书文档创建

在 Step 1 用户说完事项后，立即创建飞书文档：

### 操作步骤

```
1. 调用feishu-doc skill，创建文档
2. 文档标题：「[用户名]的事项清单」
3. 拿到 doc_token 后，调用 append 或 write action 写入内容
4. 内容写入成功后，调用 feishu-perm 给用户 full_access 权限
5. 返回文档链接给用户
```

备注：如果在写入日历或者创建文档遇到问题，需要向用户提供解决方案，解决后继续完成配置。

---

## 创建定时任务

在 Step 6 确认后，使用 OpenClaw cron 创建定时任务。

### 重要说明

定时任务的消息是发给 **OpenClaw** 的，不是直接发给用户的。OpenClaw 收到消息后，需要根据消息内容执行对齐流程并与用户交互。

因此消息内容需要：
1. 告诉 OpenClaw 现在是什么对齐时间
2. 指引 OpenClaw 按照对应的流程文档执行
3. 直接让 OpenClaw 向用户发送开场消息（不要说"收到"之类的过渡语）

### 日对齐定时任务消息模板

```bash
pnpm openclaw cron add \
  --name "日对齐" \
  --cron "30 21 * * *" \
  --tz "Asia/Shanghai" \
  --message "【日对齐提醒】

现在到每日对齐时间了。

请执行 schedule-everyday skill 的日对齐流程。

⚠️ 重要：
1. 先读取用户配置：~/.openclaw/workspace/skills/schedule-everyday/config.yaml
2. 再读取事项清单（飞书文档 token 在配置的 storage.feishu_doc_token）
3. 同时查看飞书日历，了解今天的固定安排
4. 然后按照 schedule-everyday skill 的 references/daily-sync-flow.md 执行

请直接向用户发送开场消息（不要说任何"收到""好的"之类的话）：

开场消息要基于用户具体事项来问，不要问太开放的问题。
例如：「简历写得怎么样？写出一版了吗？」或「今天主线想做什么？看你有论文要交、简历也在进行中。」" \
  --channel feishu \
  --to "ou_xxx" \
  --announce
```

### 周对齐定时任务消息模板

```bash
pnpm openclaw cron add \
  --name "周对齐" \
  --cron "0 19 * * 0" \
  --tz "Asia/Shanghai" \
  --message "【周对齐提醒】

现在到每周对齐时间了。

请执行 schedule-everyday skill 的周对齐流程。

⚠️ 重要：
1. 先读取用户配置：~/.openclaw/workspace/skills/schedule-everyday/config.yaml
2. 再读取事项清单（飞书文档 token 在配置的 storage.feishu_doc_token）
3. 同时查看飞书日历，了解本周的固定安排
4. 然后按照 schedule-everyday skill 的 references/weekly-sync-flow.md 执行

请直接向用户发送开场消息（不要说任何"收到""好的"之类的话）：

📅 周对齐时间到了。

先看中期目标 — 目前进展如何？

[根据配置中的 goals 字段展示目标表格]" \
  --channel feishu \
  --to "ou_xxx" \
  --announce
```

### cron 表达式说明

| 对齐类型 | 时间示例 | cron 表达式 |
|----------|----------|-------------|
| 日对齐 - 早上 8:00 | 每天 8:00 | `0 8 * * *` |
| 日对齐 - 晚上 21:30 | 每天 21:30 | `30 21 * * *` |
| 周对齐 - 周日晚上 | 周日 19:00 | `0 19 * * 0` |
| 周对齐 - 周一早上 | 周一 8:00 | `0 8 * * 1` |

## 配置保存

### 配置文件路径

按优先级查找/保存配置：
- **OpenClaw 环境**（推荐）：`~/.openclaw/workspace/skills/schedule-everyday/config.yaml`
- **Claude Code 环境**（fallback）：`~/.schedule-everyday/config.yaml`

### 断点续传机制

在 Onboarding 过程中，每完成一个关键步骤后立即更新配置文件中的 `onboarding_status` 字段：

```yaml
onboarding_status: "pending_step3"  # 或 pending_step4 / pending_step5 / completed
```

### 完成后保存的配置（仅示例）

```yaml
version: "1.0"
created_at: "YYYY-MM-DD"
onboarding_status: "completed"

classification: "moscow"  # moscow / 1-3-5 / eisenhower

daily_sync:
  enabled: true
  time: "21:30"

weekly_sync:
  enabled: true
  day: "sunday"
  time: "19:00"

calendar:
  enabled: true
  color_by: "domain"  # domain（领域）/ priority（优先级）/ none（不分）
  color_mapping:
    "工作": -562844      # 红色
    "学业": -7120138     # 紫色
    "内容输出": -16722247 # 青色
    "运动": -30720       # 橙色
    "琐事": -6511959     # 灰色

goals:
  - name: "目标1"
    important: true
  - name: "目标2"
    important: false

storage:
  feishu_doc_token: "xxx"
  feishu_calendar_id: "openclaw"  # 使用 OpenClaw 日历
  local_path: "~/.openclaw/workspace/skills/schedule-everyday/tasks.md"
```

---

```markdown
**主线**：（待日对齐确定）

### 今日日程

（待填充）

---

## [根据分类模板填充结构]

---

## 🎯 目标

| 目标 | 进展 |
|------|------|

---

## 已完成

（完成后会记录在这里）

---

## 归档

（不再做的事项会移到这里）
```

## 重新设置

如果用户说"重新设置"：

```
要重新设置吗？这会：
- 重置偏好设置
- 保留现有事项清单

确认吗？
- A. 确认，重新设置
- B. 算了，保持现状
```
