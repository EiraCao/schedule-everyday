# 日对齐流程

本文档描述日对齐的完整流程，包含理想案例供参考。

---

## 核心风格

对齐要**轻量自然**，不是审问，也不是清点库存。

- 每次只问一个问题
- 提供选项，但不要写成 ABCD 这种呆板格式
- 像朋友聊天一样，语气轻松
- 用户只需要"拍板"，思考和整理由你完成

---

## 流程概览

1. **回顾刚刚过去的一天** — 做了什么，有没有没完成的
2. **确定即将到来的一天的主线** — 最重要的一件事
3. **扫描临期事项** — 检查有没有快到期的事
4. **创建定时提醒** — 确认临期小事的时间，立即创建提醒
5. **确认固定安排** — 会议、课程等
6. **填充空闲时间** — 主线任务排到什么时候
7. **如果有大片空白，推荐探索类事项**

---

## 详细流程

### Step 1：回顾刚刚过去的一天

先问过去的情况，建立连续性。

**晚上日对齐**：回顾今天白天
**早上日对齐**：回顾昨天

```
刚刚过去的一天怎么样？主线完成了吗？
```

或者如果知道之前安排了什么：

```
简历写得如何？写出一版了吗？
```

### Step 2：确定即将到来的一天的主线

问最重要的一件事，同时给时间选项。

**晚上日对齐**：计划明天
**早上日对齐**：计划今天

```
即将到来的一天主线想做什么？

看你有论文要交、简历也在进行中。如果只能选一件最重要的，是哪个？

另外想给自己留多长时间？2 小时紧凑一点，3 小时宽松一点，还是写完为止？
```

### Step 3：扫描临期事项

根据用户选择的分类方式，检查对应的高优先级分类。

**MoSCoW 方式**：检查 Must 分类中临期/过期的事项
**1-3-5 方式**：检查大任务候选中临期的事项
**Eisenhower 方式**：检查 Q1（紧急且重要）

**示例**：

```
我看了一下，有几件临期的事：

- 作业邮件提交 — 昨天就该交了
- 拿快递 — 今天不拿就过期了

这两个什么时候做？
```

### Step 4：创建定时提醒

用户确认临期小事的时间后，**立即创建提醒**并确认"已设置"。

```
✅ 已设置：
- 15:30 提醒：提交作业
- 20:00 提醒：拿快递
```

### Step 5：确认固定安排

问即将到来的一天有没有固定的日程。

```
即将到来的一天有固定的安排吗？课程、会议之类的？
```

如果从日历已经能看到：

```
我看到有这些固定安排：

- 14:00-15:30 数理统计
- 16:00-18:30 公司任务

还有其他的吗？
```

### Step 6：填充空闲时间

计算剩余时间，问主线任务排到什么时候。

```
现在看剩下的时间：

- 18:30-20:00 晚饭
- 20:00 后有空

主线任务（写简历）排到什么时候？
```

### Step 7：如果有大片空白，推荐探索类事项

主线任务排完后，如果日历还有大片空白，推荐探索类事项。

根据分类方式检查对应分类：
**MoSCoW 方式**：从 Could 分类推荐
**1-3-5 方式**：从小任务候选推荐
**Eisenhower 方式**：从 Q2 推荐（长期价值任务）

```
还有一小段空着。探索类里有几个项目，要不要安排？

- 日程管理 Agent
- AI 社交产品调研
```

## 日对齐完整案例

**秘书**：

> 刚刚过去的一天怎么样？

**用户**：还行，简历写了但没写完，今天继续。

**秘书**：

> 收到。即将到来的一天继续写简历，想给自己留多长时间？
>
> - 2 小时（紧凑）
> - 3 小时（宽松）
> - 写完为止

**用户**：2 小时吧，希望可以写完一版。

**秘书**：

> 明白，写简历 2 小时，目标是写完一版。
>
> 我看了一下，有几件临期的事：
>
> - ⚠️ 作业邮件提交：昨天就该交
> - ⚠️ 拿快递：今天不拿就过期
>
> 这两个什么时候做？

**用户**：作业下课做，快递晚上拿。

**秘书**：

> ✅ 已设置：
> - 15:30 提醒：提交作业
> - 20:00 提醒：拿快递
>
> 即将到来的一天有固定安排吗？

**用户**：有，14:00-15:30 数理统计，然后要去公司 16:00-18:30。

**秘书**：

> 明白了。今天的安排：
>
> - 14:00-15:30 数理统计
> - 15:30 ⏰ 提醒：提交作业
> - 16:00-18:30 公司任务
> - 18:30-20:00 晚饭
> - 20:00 ⏰ 提醒：拿快递
>
> 剩下 20:00 后有空。写简历排到什么时候？

**用户**：21:00-23:00 吧。

**秘书**：

> ✅ 日历已更新：21:00-23:00 写简历
>
> 还有 20:30-21:00 一小段空着。探索类里有几个项目，要不要安排？

**用户**：20:30-21:00 做暂停实验室练习。

**秘书**：

> ✅ 已添加。
>
> **今天安排确认**：
>
> - 14:00-15:30 数理统计
> - 15:30 ⏰ 提醒：提交作业
> - 16:00-18:30 公司任务
> - 18:30-20:00 晚饭
> - 20:00 ⏰ 提醒：拿快递
> - 20:30-21:00 暂停实验室
> - 21:00-23:00 写简历
>
> 没问题？

**用户**：没问题了。

**秘书**：

> 好的，日对齐完成。

## 定时任务消息模板

定时任务的消息是发给 **OpenClaw** 的，不是直接发给用户的。

```bash
pnpm openclaw cron add \
  --name "日对齐" \
  --cron "30 21 * * *" \
  --tz "Asia/Shanghai" \
  --message "【日对齐提醒】

现在到每日对齐时间了。

请直接向用户发送以下内容（不要说任何"收到"之类的话）：

🌙 每日对齐时间到了。

刚刚过去的一天怎么样？

然后按照 schedule-manager skill 的 references/daily-sync-flow.md 中的流程继续。

请先读取用户的配置和事项清单：
- 配置文件路径：[用户配置路径]
- 事项清单：[飞书文档链接]" \
  --channel feishu \
  --to "ou_xxx" \
  --announce
```
